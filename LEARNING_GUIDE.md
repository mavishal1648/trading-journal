# Trading Journal — Learning Guide

A simple explanation of everything in this project, written for someone new to Prisma and Supabase.

---

## What is Supabase?

Think of Supabase as a **free backend in the cloud**. It gives you:

1. **A Database (PostgreSQL)** — where your trade data lives (like a spreadsheet but way more powerful)
2. **File Storage** — where your screenshot images are stored (like Google Drive but for your app)
3. **A Dashboard** — a website where you can see your data, manage tables, view files

### How it works in our app:
```
You add a trade → Data goes to Supabase Database (tables)
You upload a screenshot → Image goes to Supabase Storage (files)
You open dashboard → App reads data FROM Supabase and shows charts
```

### Key Supabase terms:
- **Project** — Your app's backend (you create one at supabase.com)
- **Anon Key** — A public API key (safe to use in browser, has limited access)
- **Connection String** — The address to your database (like a URL but for databases)
- **Storage Bucket** — A folder in Supabase where files (screenshots) are stored

## What is Prisma?

Prisma is a **translator between your TypeScript code and your database**.

Without Prisma, you'd write raw SQL:
```sql
SELECT * FROM trades WHERE instrument = 'NQ' AND result = 'WIN';
```

With Prisma, you write TypeScript:
```typescript
const trades = await prisma.trade.findMany({
  where: { instrument: "NQ", result: "WIN" }
});
```

Prisma converts your TypeScript into SQL behind the scenes. It also gives you **autocomplete** and **type safety** — so if you misspell a column name, your editor shows an error immediately.

### Key Prisma terms:
- **Schema** (`prisma/schema.prisma`) — A file where you define your database tables and columns. Think of it as a blueprint.
- **Model** — A table definition. `model Trade { ... }` = a table called "Trade"
- **Migration** — When you change the schema, a migration updates the actual database to match. Like syncing a blueprint to the real building.
- **Generate** — Creates TypeScript code from your schema so you get autocomplete. Run `npx prisma generate` after changing the schema.
- **Prisma Client** — The auto-generated code you import to query the database.
- **Adapter** — In Prisma 7, you need an adapter to connect to your database. We use `@prisma/adapter-pg` for PostgreSQL/Supabase.

### Prisma workflow:
```
1. Edit prisma/schema.prisma (add/change tables)
2. Run: npx prisma generate (generates TypeScript types)
3. Run: npx prisma migrate dev --name "what_you_changed" (updates the actual database)
4. Use prisma.trade.findMany(), prisma.trade.create(), etc. in your code
```

## How Our Database is Structured

We have 4 tables. Here's how they connect:

```
┌─────────────┐       ┌──────────────┐
│   Trade     │──────▶│  Screenshot  │
│             │ 1 : N │              │
│ id          │       │ id           │
│ instrument  │       │ tradeId (FK) │  ← Points back to Trade
│ direction   │       │ url          │  ← Just a link to the image file
│ entryPrice  │       └──────────────┘
│ exitPrice   │
│ quantity    │       ┌──────────────┐      ┌──────────┐
│ riskReward  │──────▶│  TradeTag    │◀─────│   Tag    │
│ result      │ N : N │              │      │          │
│ pnl         │       │ tradeId (FK) │      │ id       │
│ rating      │       │ tagId   (FK) │      │ name     │
│ notes       │       └──────────────┘      │ color    │
│ tradeDate   │                             └──────────┘
└─────────────┘
```

### Relationships explained:
- **Trade → Screenshot** (One-to-Many): One trade can have up to 2 screenshots. Each screenshot belongs to one trade.
- **Trade → Tag** (Many-to-Many): One trade can have multiple tags. One tag can be on multiple trades. The `TradeTag` table connects them (called a "join table").
- **FK = Foreign Key**: A column that points to another table's ID. Like a reference/link.

### What happens when you delete a trade?
- `onDelete: Cascade` means: if you delete a trade, its screenshots and tag connections are automatically deleted too. No orphan data.

## Key Concepts

### What is a "Singleton"?
In `prisma.ts`, we create ONE database connection and reuse it everywhere. Without this, every page request would create a NEW connection, which wastes resources and can crash your database.

```typescript
// BAD: Creates a new connection every time
const prisma = new PrismaClient(); // Don't do this in every file!

// GOOD: Create once, reuse everywhere (what we do)
const globalForPrisma = globalThis as unknown as { prisma: PrismaClient };
export const prisma = globalForPrisma.prisma || new PrismaClient({ ... });
```

### What is a "Server Action"?
In Next.js, a server action is a function that runs on the SERVER, not in the browser. We use these for all database operations because:
- The database connection string is secret (can't expose it to the browser)
- SQL queries should run server-side for security

```typescript
// This function runs on the server
"use server";

export async function createTrade(data: TradeFormData) {
  await prisma.trade.create({ data: { ... } });
}
```

### What is an "Adapter" (Prisma 7)?
Prisma 7 removed the old `datasourceUrl` option. Now you need an "adapter" — a small package that knows how to talk to your specific database type:
- `@prisma/adapter-pg` — for PostgreSQL (what Supabase uses)
- There are also adapters for MySQL, SQLite, etc.

Think of it like a USB adapter — Prisma is the laptop, PostgreSQL is the device, the adapter connects them.

### What is WebP conversion?
When you upload a screenshot (PNG/JPG), we convert it to WebP format in your browser BEFORE uploading. WebP is a modern image format that's ~70% smaller. A 1MB PNG becomes ~100KB WebP. This saves storage space on Supabase's free 1GB tier.

## Setting Up Supabase (when you're ready)

1. Go to [supabase.com](https://supabase.com) and create a free account
2. Create a new project (choose a region close to you)
3. Go to **Project Settings → Database** and copy the connection string
4. Go to **Project Settings → API** and copy the URL + anon key
5. Paste them into `.env.local`:
   ```
   DATABASE_URL="postgresql://postgres:YOUR_PASSWORD@db.YOUR_PROJECT.supabase.co:5432/postgres"
   DIRECT_URL="postgresql://postgres:YOUR_PASSWORD@db.YOUR_PROJECT.supabase.co:5432/postgres"
   NEXT_PUBLIC_SUPABASE_URL="https://YOUR_PROJECT.supabase.co"
   NEXT_PUBLIC_SUPABASE_ANON_KEY="your-anon-key-here"
   ```
6. Run `npx prisma migrate dev --name "init"` to create the tables
7. In Supabase dashboard, go to **Storage** and create a bucket called `screenshots`
